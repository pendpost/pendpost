# Support

Thanks for using pendpost. Here is where to get help.

## Documentation

Start with the docs at https://docs.pendpost.com and the quickstart in the README. Most questions are answered there. You do not need any credentials to try pendpost: mock mode runs the full draft to approve to schedule to publish to insights loop with zero credentials.

## Questions and ideas

For usage questions, setup help, and ideas, use GitHub Discussions at https://github.com/pendpost/pendpost/discussions. This is the best place for open-ended conversation.

## Bugs and feature requests

If you have found a bug or want to request a feature, open an issue and pick the bug report or feature request form. The form prompts you for the details we need to act on it.

## Security issues

Do not open a public issue for a security vulnerability. See SECURITY.md for how to report privately. You can email security@pendpost.com or open a private security advisory on GitHub.

## Response time

pendpost is maintained part-time by a small team, so please allow time for a response. Clear, focused, reproducible reports are handled fastest.
